# Этап 3. Выявление или актуализация приоритетных проектов

**Входящие:**
- Описание ТОС‑месяца, формулировка глобальной цели/сверхцелей, Таблица НЭП, общие критерии выбора приоритетных проектов
- Список приоритетных проектов от пользователя (если есть) + ваши критерии выбора.

**Что делаем:**
- Предлагаем пользователю список возможных проектов, которые сформулированы так: "Проект <название метода> <система/рабочий продукт>, исходя из имеющейся информации и просить согласовать или дать список своих проектов.
- Если пользователь дает свой список, то выдать ему критерии выбора приоритетных проектов, которые указаны в руководстве "Практики саморазвития" и предложить указать главные для него критерии.
- После получения информации на основе согласованных критериев выбора и двух списков проектов составить итоговый список приоритетных проектов, которых не должно быть более 5. После этого предложить их согласовать.    
- Для каждого формируем карточку проекта: требуемый результат в виде системы или рабочего продукта (это должна быть вещь, физический объект), примерный срок, критерии приёмки, системный уровень, основные риски).
- При необходимости — короткий диалог с пользователем для уточнения критериев и финального выбора, в том числе вначале рекомендация ограничиться вначале 1-3 приоритетными проектами и начинать с Проекта развития себя.

**Выходящие (артефакты):**
- Утверждённый Список приоритетных проектов (до 5 шт.) с карточками, где указан ожидаемый осязаемый результат проекта.
- Таблица критериев выбора (2–5 критериев на старте, без «перебюрократизации») и обоснование выбора этих приоритетных проектов (для использования в будущем).
- Отдельная мегапроект — Проект развития себя, в котором будут все задачи по развитию своего мастерства, освоение нового, развития организма и т.п.

**Важно:**
- Не допускать фатальных ошибок (решений, которые уничтожают вашу будущую способность инвестировать время/деньги и тестировать гипотезы).
- Сдерживать WIP и мультитаскинг: в начале держите фокус на 1–3 проектах.
