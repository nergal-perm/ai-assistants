# Выявление приоритетных проектов

## Обзор

Этап определения наиболее влиятельных точек приложения усилий. На основе ТОС-месяца и глобальной цели формируется список приоритетных проектов (до 5) с карточками, где для каждого проекта указан ожидаемый осязаемый результат.

## Параметры

- **тос_месяц** (обязательно): Описание ТОС‑месяца с этапа 2
- **глобальная_цель** (обязательно): Формулировка глобальной цели/сверхцелей
- **таблица_нэп** (обязательно): Таблица неудовлетворенностей, эмоций и проблем
- **существующие_проекты** (не обязательно): Список приоритетных проектов от пользователя
- **критерии_выбора** (не обязательно): Общие критерии выбора приоритетных проектов пользователя

**Ограничения для получения параметров:**

- Ты ДОЛЖЕН использовать ТОС-месяца и глобальную цель для определения приоритетов
- Ты МОЖЕШЬ запросить у пользователя существующий список проектов
- Тебе РЕКОМЕНДУЕТСЯ использовать критерии выбора из руководства «Практики саморазвития»

## Шаги

### 1. Предложение списка возможных проектов

Сформируй предварительный список проектов на основе анализа предыдущих этапов.

**Ограничения:**

- Ты ДОЛЖЕН формулировать проекты в формате: "Проект <название метода> <система/рабочий продукт>"
- Ты ДОЛЖЕН предложить пользователю согласовать список или дать свой собственный
- Ты ДОЛЖЕН совместно с пользователем выделить несколько наиболее влиятельных узлов или привычек, где малые изменения могут дать заметный эффект

### 2. Согласование критериев выбора

Определи критерии приоритизации проектов совместно с пользователем.

**Ограничения:**

- Ты ДОЛЖЕН выдать критерии выбора приоритетных проектов из руководства «Практики саморазвития», если пользователь дает свой список
- Ты ДОЛЖЕН предложить указать главные для пользователя критерии
- Ты ДОЛЖЕН составить таблицу критериев выбора (2–5 критериев на старте, без «перебюрократизации»)
- Тебе РЕКОМЕНДУЕТСЯ ограничиться вначале 1-3 приоритетными проектами
- Тебе РЕКОМЕНДУЕТСЯ начинать с Проекта развития себя

### 3. Формирование итогового списка проектов

Составь финальный список приоритетных проектов на основе согласованных критериев.

**Ограничения:**

- Ты ДОЛЖЕН на основе согласованных критериев выбора и двух списков проектов составить итоговый список
- Ты ДОЛЖЕН ограничить список максимум 5 проектами
- Ты ДОЛЖЕН предложить согласовать итоговый список
- Тебе НЕ СЛЕДУЕТ включать более 5 проектов одновременно

### 4. Создание карточек проектов

Для каждого приоритетного проекта сформируй детальную карточку.

**Ограничения:**

- Ты ДОЛЖЕН для каждого проекта создать карточку со следующими полями:
  - Требуемый результат в виде системы или рабочего продукта (это должна быть вещь, физический объект)
  - Примерный срок
  - Критерии приёмки
  - Системный уровень
  - Основные риски
- Ты ДОЛЖЕН обеспечить, чтобы ожидаемый результат был осязаемым и конкретным

### 5. Выделение мегапроекта развития себя

Определи или актуализируй отдельный метапроект по саморазвитию.

**Ограничения:**

- Ты ДОЛЖЕН создать отдельный мегапроект — Проект развития себя
- Ты ДОЛЖЕН включить в этот проект все задачи по развитию мастерства, освоению нового, развитию организма и т.п.

## Примечания

**Важные принципы:**

- Не допускай фатальных ошибок (решений, которые уничтожают будущую способность инвестировать время/деньги и тестировать гипотезы)
- Сдерживай WIP (work in progress) и мультитаскинг: в начале держи фокус на 1–3 проектах
- Сохраняй обоснование выбора приоритетных проектов для использования в будущем
