# Диагностика узкого места

## Обзор

Этот этап посвящен системному анализу ситуации для выявления корневых факторов и ключевых узких мест на ближайший период (минимум месяц). Проводится «четырёхмерный» анализ bottleneck по методу Голдрата, адаптированному к личному потоку.

## Параметры

- **таблица_нэп** (required, assistant_created): Таблица неудовлетворенностей, эмоций и проблем с этапа 1
- **глобальная_цель** (optional, user_input): Наброски глобальной цели или сверхцелей пользователя
- **данные_цифрового_двойника** (optional, user_input): Информация о ролевой позиции на траектории (Ученик → Интеллектуал → Профессионал → Исследователь), шкала самооценки состояния (1–6)

**Ограничения для получения параметров:**

- Ты ДОЛЖЕН использовать таблицу НЭП, сформированную на этапе 1
- Ты МОЖЕШЬ запросить дополнительную информацию о глобальной цели, если она недостаточно четкая
- Тебе РЕКОМЕНДУЕТСЯ использовать данные цифрового двойника, если они доступны

## Шаги

### 1. Самооценка по шкале застревания

Предложи пользователю провести самооценку по шкале застревания 1-6.

**Ограничения:**

- Ты ДОЛЖЕН объяснить шкалу самооценки состояния (1–6) о том, где застрял человек
- Ты ДОЛЖЕН запросить текущую оценку у пользователя
- Ты МОЖЕШЬ использовать предыдущую оценку для сравнения, если она доступна

### 2. Четырёхмерный анализ bottleneck

Проведи системный анализ узкого места по четырём измерениям.

**Ограничения:**

- Ты ДОЛЖЕН учитывать актуальные проблемы/неудовлетворённости и шкалу застревания
- Ты ДОЛЖЕН учитывать ролевую траекторию (если доступна)
- Ты ДОЛЖЕН учитывать объекты внимания «день‑неделя‑месяц»
- Ты ДОЛЖЕН применять принципы выбора методов: простые→сложные, когнитивная экономия, ритм>объём, лестница освоения
- Ты ДОЛЖЕН помочь пользователю увидеть ситуацию как систему: участники, роли, ресурсы, повторяющиеся паттерны, связи, обратные связи, ограничения
- Ты ДОЛЖЕН отделять поверхностные симптомы от более глубоких механизмов и условий
- Тебе РЕКОМЕНДУЕТСЯ аккуратно проверять выводы вопросами и альтернативными гипотезами

### 3. Классификация ситуации и формулировка ТОС-месяц

Классифицируй ситуацию и сформируй карточку ТОС-месяца (теория ограничений систем).

**Ограничения:**

- Ты ДОЛЖЕН классифицировать ситуацию как: Хаос / Тупик / Поворот / Что-то другое
- Ты ДОЛЖЕН подобрать соответствующие интервенции:
  - Хаос → стабилизация
  - Тупик → пробитие потолка
  - Поворот → управляемый разворот
- Ты ДОЛЖЕН сформулировать ТОС‑месяц (одно доминирующее ограничение)
- Ты ДОЛЖЕН предложить гипотезу снятия узкого места
- Ты ДОЛЖЕН определить критерий успеха
- Ты ДОЛЖЕН идентифицировать риски и составить стоп‑лист

### 4. Формирование карты ограничений

Создай структурированное описание стратегии работы с узким местом.

**Ограничения:**

- Ты ДОЛЖЕН создать карту ограничений, включающую:
  - Что держим неизменным
  - Что меняем первым
  - Риски и стоп‑лист
- Ты ДОЛЖЕН обновить самооценку 1–6 с целевым трендом на конец месяца
- Ты ДОЛЖЕН составить карточку ТОС‑месяца с формулировкой узкого места, гипотезой снятия и метриками

## Примечания

**Важные принципы:**

- Узкое место определяется только в контексте текущего потока (не «вообще»)
- Стартуй с минимально достаточных методов и обратимых шагов
- Удерживай принцип: ритм>объём
- Веди логи в экзокортексе (решения, источники, артефакты) — без логов перенос знаний слабеет
