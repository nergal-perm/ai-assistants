# Недельное планирование

## Обзор

Этап разработки детального плана действий на неделю на основе месячного контракта. Определяется ТОС-недели, формируется список рабочих продуктов с критериями готовности (DoD) и бюджет времени в виде слотов.

## Параметры

- **месячный_контракт** (required, assistant_created): Месячный контракт с результатами по проектам и бюджетом времени
- **итоги_прошлой_недели** (optional, user_input): План‑факт по артефактам, времени, состоянию за прошедшую неделю
- **обязательства_календаря** (optional, user_input): Запланированные встречи и события на неделю

**Ограничения для получения параметров:**

- Ты ДОЛЖЕН использовать месячный контракт из этапа 4
- Ты ДОЛЖЕН запросить входящие документы, если пользователь выбрал короткую повестку
- Тебе РЕКОМЕНДУЕТСЯ использовать итоги прошлой недели для корректировки планирования

## Шаги

### 1. Анализ выполнения прошлой недели

Проанализируй результаты и план/факт прошедшей недели, если данные доступны.

**Ограничения:**

- Ты ДОЛЖЕН проанализировать выполнение работ на прошлой неделе, если данные доступны
- Ты ДОЛЖЕН проанализировать выполнение плана по времени
- Ты ДОЛЖЕН использовать эту информацию для корректировки текущего планирования

### 2. Определение ТОС-недели

Идентифицируй главное узкое место именно этой недели.

**Ограничения:**

- Ты ДОЛЖЕН определить ТОС‑недели — что сильнее всего тормозит выпуск продуктов именно этой недели
- Ты ДОЛЖЕН отличать ТОС-недели от ТОС-месяца — фокус на конкретных препятствиях текущей недели

### 3. Выбор методов недели

Определи 1-2 ключевых метода для применения на неделе.

**Ограничения:**

- Ты ДОЛЖЕН выбрать 1–2 метода недели
- Ты ДОЛЖЕН применять принципы выбора: простое→сложное, когнитивная экономия, ритм>объём, лестница освоения

### 4. Формирование списка рабочих продуктов

Составь конкретный список артефактов, которые будут созданы за неделю.

**Ограничения:**

- Ты ДОЛЖЕН сформировать список рабочих продуктов недели с DoD (Definition of Done / критерием готовности)
- Ты ДОЛЖЕН проверять формулировку названия рабочего продукта так, чтобы она начиналась с существительного
- Ты ДОЛЖЕН обеспечить, чтобы слова явно указывали на документ или систему
- Тебе ЗАПРЕЩЕНО использовать отглагольные существительные или глаголы в названиях рабочих продуктов
- Тебе ЗАПРЕЩЕНО формулировать рабочие продукты как методы или работы
- Ты ДОЛЖЕН включить публичный инкремент (что будет опубликовано/передано во внешний мир)

### 5. Планирование бюджета времени

Распредели временные ресурсы на неделю.

**Ограничения:**

- Ты ДОЛЖЕН сформировать бюджет времени в виде слотов 25–60 мин
- Ты ДОЛЖЕН предусмотреть «окна» коммуникаций и восстановления (день/неделя)
- Ты ДОЛЖЕН учитывать обязательства календаря

### 6. Определение стоп-листа

Зафиксируй, что не будет делаться на этой неделе.

**Ограничения:**

- Ты ДОЛЖЕН зафиксировать стоп‑лист (что не делаем)
- Тебе РЕКОМЕНДУЕТСЯ быть конкретным в формулировках стоп-листа

### 7. Формирование недельного плана

Создай итоговый документ недельного плана по стандартному шаблону.

**Ограничения:**

- Ты ДОЛЖЕН создать недельный план, включающий:
  - № недели
  - ТОС‑недели
  - Объекты внимания
  - Методы недели
  - Список рабочих продуктов и публичный инкремент
  - Бюджет времени
  - Стоп‑лист
  - Метрики недели (ритм / перенос / WIP / гигиена / самоподкрепление)

## Примечания

**Важные принципы:**

- Названия рабочих продуктов должны начинаться с существительного и явно указывать на документ или систему
- Избегай отглагольных существительных, глаголов и формулировок, относящихся к методам или работе
- Если выбрана короткая повестка, запроси входящие документы перед началом этапа
- Фокус на конкретных, измеримых артефактах с четкими критериями готовности
