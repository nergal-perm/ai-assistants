# Определение повестки сессии

## Обзор

На этом этапе устанавливается контекст и повестка текущей сессии стратегирования. Определяется, будет ли проведена короткая или полная (длинная) сессия, а также анализируется структура хранилища пользователя для определения местоположения необходимых заметок.

## Параметры

- **запрос_на_сессию** (обязательно): Запрос пользователя на проведение сессии стратегирования
- **предыдущие_сессии** (не обязательно): Записи прошлых сессий стратегирования
- **структура_хранилища** (не обязательно): Сохраненная информация о расположении заметок в vault

## Шаги

### 1. Выбор формата повестки

Предложи пользователю выбрать между короткой и длинной повесткой сессии стратегирования.

**Ограничения:**

- Ты ДОЛЖЕН объяснить различия между короткой и длинной повесткой
- Ты ДОЛЖЕН указать, что короткая повестка включает только этапы 1, 5, 6
- Ты ДОЛЖЕН указать, что длинная повестка включает все этапы с 1 по 6
- Ты ДОЛЖЕН запросить согласование повестки или предложить ее скорректировать

**Состав короткой повестки:**

1. **Этап 1.** Определение неудовлетворенностей, эмоций, проблем [[stage-1-nep.sop.md]] — формирование таблицы НЭП
2. **Этап 5.** Недельное планирование [[stage-5-weekly-plan.sop.md]] — определение ТОС-недели и списка рабочих продуктов
3. **Этап 6.** Установки на дневное исполнение [[stage-6-daily-execution.sop.md]] — рекомендации по слот-дисциплине

**Длинная повестка включает все этапы:**

1. **Этап 1.** Определение неудовлетворенностей, эмоций, проблем [[stage-1-nep.sop.md]]
   - Разобрать дневниковые записи, мимолетные заметки, просмотреть все ранее сформированные черновики
   - Прочитать и скорректировать список проблем и неудовлетворенностей с предыдущих сессий стратегирования
   - Сформировать таблицу НЭП (Неудовлетворенность, Эмоции, Проблемы)

2. **Этап 2.** Диагностика узкого места [[stage-2-bottleneck.sop.md]]
   - Определить или скорректировать глобальную цель
   - Определить ТОС-месяца и гипотезы его снятия

3. **Этап 3.** Выявление приоритетных проектов [[stage-3-projects.sop.md]]
   - Определить или скорректировать список приоритетных проектов
   - При необходимости открыть новый проект
   - Составить список до 5 проектов с карточками

4. **Этап 4.** Составление месячного контракта [[stage-4-contract.sop.md]]
   - Определить или скорректировать месячный контракт

5. **Этап 5.** Недельное планирование [[stage-5-weekly-plan.sop.md]]
   - Проанализировать выполнение работ на прошлой неделе и выполнение плана по времени
   - Определить ТОС текущей недели
   - Составить недельный план работ, список рабочих продуктов на неделю и бюджет времени

6. **Этап 6.** Установки на дневное исполнение [[stage-6-daily-execution.sop.md]]
   - Рекомендации по слот-дисциплине

### 2. Анализ структуры хранилища

Проанализируй структуру хранилища (vault) пользователя и определи расположение заметок различных типов, необходимых для проведения сессии.

**Ограничения:**

- Ты ДОЛЖЕН проводить анализ структуры только один раз в начале первой сессии или при существенных изменениях в организации хранилища
- Ты ДОЛЖЕН использовать сохраненный Knowledge Item в последующих сессиях
- Ты ДОЛЖЕН обновить Knowledge Item, если пользователь сообщает об изменениях в структуре

**Типы заметок для всех типов сессий:**

1. Дневниковые записи / утренние страницы — ежедневная рефлексия, мимолетные заметки
2. Черновики и заготовки — незавершенные записи о проблемах и неудовлетворенностях
3. Недельные планы — план рабочих продуктов и бюджет времени
4. Результаты прошедшей недели — план-факт по артефактам, времени, состоянию

**Типы заметок для длинной повестки дополнительно:**
5. Таблица НЭП — предыдущие таблицы неудовлетворенностей, эмоций и проблем
6. Формулировка глобальной цели — наброски сверхцелей и целей
7. Данные цифрового двойника — информация о ролевой позиции на траектории, шкала самооценки
8. Список приоритетных проектов — текущие и предыдущие проекты
9. Критерии выбора проектов — критерии приоритизации проектов
10. Месячные/квартальные контракты — предыдущие контракты с результатами
11. Список литературы для чтения — книги и статьи для медленного чтения
12. Обязательства календаря — запланированные встречи и события
13. Артефакты рабочих продуктов — созданные документы и системы

### 3. Определение местоположения заметок

Создай структурированное описание расположения типов заметок в хранилище.

**Ограничения:**

- Ты ДОЛЖЕН просмотреть все основные папки и подпапки
- Ты ДОЛЖЕН обратить внимание на названия папок, которые могут указывать на содержимое
- Ты ДОЛЖЕН изучить несколько файлов из разных папок для понимания их содержания
- Ты ДОЛЖЕН для каждого типа заметок указать папку или папки, где они хранятся
- Ты ДОЛЖЕН явно отметить, если какой-то тип заметок не найден
- Ты ДОЛЖЕН обратить внимание на паттерны именования файлов

### 4. Сохранение структуры в Knowledge Item

Сохрани информацию о структуре хранилища для использования в будущих сессиях.

**Ограничения:**

- Ты ДОЛЖЕН создать или обновить Knowledge Item с названием "Vault Structure - Strategy Sessions"
- Ты ДОЛЖЕН сохранить полный mapping типов заметок и их расположения
- Ты ДОЛЖЕН указать дату последнего обновления структуры

### 5. Подтверждение структуры пользователем

Представь пользователю предложенную структуру хранилища и запроси подтверждение.

**Ограничения:**

- Ты ДОЛЖЕН представить структуру в виде таблицы с колонками: Тип заметок, Расположение, Примеры файлов, Примечания
- Ты ДОЛЖЕН попросить подтвердить правильность определения расположения заметок
- Ты ДОЛЖЕН скорректировать mapping и обновить Knowledge Item, если пользователь указывает на ошибки или уточнения
- Ты ДОЛЖЕН после согласования перейти к следующему этапу выбранной повестки

**Пример формата представления структуры:**

| Тип заметок        | Расположение        | Примеры файлов                   | Примечания                     |
| ------------------ | ------------------- | -------------------------------- | ------------------------------ |
| Дневниковые записи | `/Daily Notes/`     | `2025-01-15.md`, `2025-01-16.md` | Ежедневные файлы с датой       |
| Недельные планы    | `/Planning/Weekly/` | `Week-03-2025.md`                | Шаблон: Week-XX-YYYY           |
| Проекты            | `/Projects/`        | `AI-Assistant.md`, `Health.md`   | Каждый проект - отдельный файл |
| Глобальные цели    | `/Goals/`           | `2025-goals.md`                  | Годовые и долгосрочные цели    |
